ZigZag Conversion

发表于 2014-12-19   |   分类于 LeetCode
最初的想法是模拟，按照顺序填充nRowslen(s)的矩阵，后来发现

把2nRows-2个元素作为一个部分，可以把蛇形分为若干个部分

每个部分中有 num = 2nRows-2 个元素，编号为0~2nRows-2 -1

对于输出结果中的第i行（从0开始）


有两类元素:

1.序号 % num = i or = 2nRows - 2 - i（后一种可以理解为倒着数）

这样可以直接知道每一行中的元素

1、没有考虑只有一行的情况，被0除了

2、def convert(s, nRows):

if nRows == 1:

return s

answer = “”

num = 2 nRows - 2

for i in range(0, nRows):

line = [x for j, x in enumerate(s) if j % num == i or j % num == (2 nRows - 2 - i)]

answer += “”.join(line)

return answer

时间复杂度O(nRows len(s))超时

看来需要把复杂度优化到O(len(s))，试想能不能直接算出结果，而不是由一个矩阵转换出来。

结果字符串分为三个部分

第一行有 len(s) / num 个元素

分别是 s[0] s[num] s[2num]…

最后一行有 ( len(s) - nRows + 1) / num 个元素

s[nRows -1] s[num+ nRows -1] s[2num + nRows -1]…


第i（1~ nRows-2）行有2(len(s) - i)/num个元素

s[i] s[num -i] s[num+i] s[num + num - i]…



排在一起

s[0] s[num] s[2num]…s[i] s[num -i] s[num+i] s[num + num - i]…s[nRows -1] s[num+ nRows -1] s[2*num + nRows -1]…


AC


看了题解。。直接模拟最简单。。多占一点空间
